# RecallAR — Offline Familiar-Person Recognition

A hackathon prototype for recognising a small whitelist of pre-authorised contacts
(family members, caregivers) from a face image. Designed for eventual integration
into a Lens Studio AR overlay.

This is **not** open-world face recognition. It compares a test face exclusively
against a registered database and returns the best match or `"Unknown"`.
When in doubt, it returns `"Unknown"` — false positives are worse than false negatives
in a care setting.

## Setup

**Python 3.10 or later** is required.

```bash
# 1. Create a virtual environment (recommended)
python -m venv .venv
source .venv/bin/activate       # Windows: .venv\Scripts\activate

# 2. Install dependencies
pip install -r requirements.txt
```

> First run will download the Facenet weights (~90 MB) automatically via DeepFace.

## Quick start

### 1. Prepare your dataset

Create a `data/` folder at the project root.
Add one sub-directory per person with their photos:

```
data/
  Anna/
    anna1.jpg
    anna2.jpg
  David/
    david1.jpg
  Emma/
    emma1.jpg
```

One clear, near-frontal photo per person is the minimum.
More photos give more robust embeddings and enable threshold evaluation.

### 2. Build the embeddings database

```bash
python src/build_embeddings.py
```

With custom paths or model:

```bash
python src/build_embeddings.py \
    --data_dir data/ \
    --output outputs/embeddings.json \
    --model Facenet \
    --detector retinaface
```

This produces `outputs/embeddings.json`:

```json
{
  "Anna":  [[0.12, -0.04, ...], [0.11, -0.03, ...]],
  "David": [[0.88,  0.02, ...]],
  "Emma":  [[0.33,  0.51, ...]]
}
```

### 3. Evaluate and choose a threshold

```bash
python src/evaluate_threshold.py
```

With a custom database path or candidate list:

```bash
python src/evaluate_threshold.py \
    --db outputs/embeddings.json \
    --threshold_candidates 0.55 0.60 0.65 0.70 0.75
```

Example output:

```
=== Pair Similarity Statistics ===

  Positive pairs (same identity) (3 pair(s)):
    mean = 0.8412   min = 0.8102   max = 0.8640

  Negative pairs (different identities) (6 pair(s)):
    mean = 0.3217   min = 0.2401   max = 0.3812

=== Threshold Evaluation (FP = impostor accepted, FN = genuine rejected) ===

  Threshold     FP Rate     FN Rate  Notes
  ----------------------------------------------------------
       0.65      0.0000      0.0000  <-- zero FP
       0.60      0.0000      0.0000  <-- zero FP

[SUGGESTED THRESHOLD] 0.65
  (prioritises low false positives — lower the value to reduce
   missed recognitions at the cost of more false acceptances)
```

### 4. Recognise a test image

```bash
python src/recognize_face.py --image test_images/test1.jpg
```

With all options explicit:

```bash
python src/recognize_face.py \
    --image test_images/test1.jpg \
    --db outputs/embeddings.json \
    --threshold 0.65
```

Output when a match is found:

```
[RESULT] MATCH    : Anna
[RESULT] Score    : 0.8312  (threshold = 0.65)
```

Output when no match:

```
[RESULT] UNKNOWN  (best score = 0.3101, threshold = 0.65)
```

Exit codes: `0` = known person matched, `1` = Unknown, `2` = processing error.

## Error handling

| Situation | Behaviour |
|-----------|-----------|
| Image file missing | `FileNotFoundError` message, graceful exit |
| Image unreadable / corrupt | `ValueError` message, graceful exit |
| No face detected | `ValueError: No face detected in: ...` |
| Multiple faces detected | `ValueError: Expected 1 face, found N in: ...` |
| Embedding extraction fails | `RuntimeError` message, graceful exit |

## Swapping the embedding backend

All embedding logic is isolated in `src/embedder.py` behind the `FaceEmbedder`
abstract base class. The rest of the pipeline (`utils.py`, `build_embeddings.py`,
`recognize_face.py`) depends only on the `FaceEmbedder` interface, never on
DeepFace directly.

To change the model, pass `--model` on the CLI or edit `DEFAULT_MODEL` in `src/config.py`:

```bash
# Higher-dimensional, generally more accurate
python src/build_embeddings.py --model Facenet512

python src/recognize_face.py --image test_images/test1.jpg --model Facenet512
```

Supported DeepFace models: `Facenet` (128-d), `Facenet512` (512-d), `ArcFace` (512-d),
`VGG-Face`, `SFace`.

To plug in a completely different library (e.g. InsightFace or an ONNX model),
subclass `FaceEmbedder` in `src/embedder.py` and pass an instance to the pipeline —
no other file needs to change.

## Exporting for Lens Studio

`outputs/embeddings.json` is already portable JSON.
To produce a simplified version with one representative embedding per person:

```python
import json, numpy as np

with open("outputs/embeddings.json") as f:
    db = json.load(f)

# Use the mean embedding as the single representative vector
simplified = {name: list(np.mean(embs, axis=0)) for name, embs in db.items()}

with open("outputs/embeddings_lens.json", "w") as f:
    json.dump(simplified, f)
```

## Adjusting the threshold

| Threshold | Effect |
|-----------|--------|
| Higher (e.g. 0.80) | Fewer false positives — stricter matching, more Unknowns |
| Lower  (e.g. 0.50) | Fewer false negatives — more matches, risk of impostors |

For a care/AR setting, **err on the side of higher thresholds** to avoid
misidentifying a stranger as a known caregiver.

Run `evaluate_threshold.py` with your actual dataset to pick the right value.
